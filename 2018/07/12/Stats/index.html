Data Objects in R

Meta-information

  • Data dictionary that describes each variable, its potential/expected values, its mapping to meaningful labels (if applicable), and its derivation (if applicable)
  • Description of the study that generated the data set
  • Description of connection/link between multiple data sets from the same study
  • Programs used to process and work with the data set

Objects

Think of data objects as a container plus the elements that fill the container.
For example, consider a set of numeric values stored as a matrix:

  • Elements = Numeric data values (mode)
  • Container = Matrix (class)
  • Object = Numeric Matrix

Modes

  • Numerical: integer, double
  • Character: quoted by ‘ ‘
  • Factor: char with class, does NOT have ‘ ‘ when printed
  • Logical: TRUE, FALSE
    • availabel operators: <, <=, >, >=, ==, !=, !, &, |
    • T and F are shorthand for the logical values TRUE and FALSE, respectively. However, they are NOT reserved strings in R and, as such, can be overwritten by the user.

Classes

Vector [1D, same mode]

Create

  • c(A, B, C, D, E). A-E could be value/vector
  • create a vector containing B copies of the vector A: rep(A, B)
  • numeric-only
    • create a numeric vector containing all of the integers within [A, B]: A:B
    • create a numeric vector containing all numbers within [A, B] with a caliper of C: seq(A, B, C)

Indexing

  • numeric
    • start with 1, not 0
    • negative index: drop the value stored in that position. V[-1] will return a length(V)-1 vector since the 1st value stored in V was dropped.
    • vector as index:V[c(5,1)] will return the data values stored in the 5th and 1st position of V
    • out of bound index: return NA
  • logical
    • return values stored in the 2nd position of the vector V: V[c(F,T,F,F,F)]
    • useful when selecting values from one object based on values in another object

Length: length()

Matrix [2D, same mode]

Create

  • Reshape vector A into a matrix with B rows and C columns: matrix(A, nrow=B, ncol=C)
    Note the sequence of number:

  • combine vectors into matrix

    • by row: rbind(x, y, z)
    • by column: cbind()
      The names of the vector objects used in the rbind() or the cbind() function call are inherited as the row or column names.
  • name of dimensions
    • add name
      rownames(myMatrix) <- c(‘row1’, ‘row2’)
      colnames(myMatrix) <- c(‘col1’, ‘col2’, ‘col3’, ‘col4’)
    • delete name:
      rownames(myMatrix) <- NULL
      colnames(myMatrix) <- NULL
  • diagnal matrix

Indexing

  • Numeric

    • select a row/column: A[B, ], A[, C]

    • negative index

      • out of bound index: return error

  • Name

  • Logical

Dimension: dim()
Matrix multiplication: %%
Element-wise multiplication:

Array [>=3D, same mode]

Create

  • reshape the vector A into D matrices with B rows and C columns
    array(A, dim=(B, C, D), dimnames = list(row.names, col.names, mat.names))
    • If the length of vector A is less than B*C*D, the elements of A will be re-cycled when filling the elements of the matrix object.

Indexing
Numeric, logical, and name indexing work in exactly the same way for the array objects as they do for matrix objects. The only difference being a third dimension needs to be specified for arrays.

  • Numeric
    Note the reshaping after slicing:

  • Name

  • Logical

Dimension: dim()

Data Frame [2D, different mode]

Comparable with matrix.

Create

  • create a data frame object of 3 columns, one for each of the vectors objects A, B, and C, and the number of rows equal to the length of the 3 vectors: data.frame(A, B, C)

    Note that char values does NOT have ‘’ when appeared in data frame

  • name of dimensions: colnames(myDf) or names(myDf)
    • return a char vector
    • rename: names(Data1)[names(Data1)==’Gender’] <- ‘Sex’

Indexing

  • Numeric, Logical, Name:
  • similar to matrix
  • out of bound index: return NULL
  • \$ operator: access a column D1$C1. This is equal to D1[, C1]

check the mode of the variables: str()

List [1D, different mode( for values)/class(for objects)]

Comparable with vector.

Create

  • a list object with 3 elements, one for each of the objects A, B, and C: list(name=A, name2=B, name3=C)

Indexing

  • [[]]: select one element at a time from a list

  • []: select multiple elements from a list

  • Numeric

  • Name: L[[“Num.Mat”]], L[c(‘Log.Vec’, ‘Chr.Vec’)]

  • Logical: only be used with ‘[ ]’
  • \$ operator: extract an object from the list L$Chr.Vec

Length: length()

Recycling & Reshaping

Naming

  • A name cannot begin with a number, and it cannot contain any of the following special characters: ^, !, $, @, +, -, /, or *
  • R is case- and spelling-sensitive.

Other things to be noticed

  • comment operator: #
  • R is a vectorized language.
    In11*1:5, the operation 11* was automatically applied element-wise to each value in the vector 1:5 without having to write a looping command. In SEX == ‘Male’, the operation == ‘Male’ was applied to the vector SEX element-wise.

Functions

A function is just a collection of R commands that have been stored in an object and that can be executed sequentially by calling the name of the function instead of the individual commands.

  • argument: things inside the ()

Useful function

  • remove all objects in the workspace: rm(list=ls()).
  • determine a data object’s mode/class: mode(), class()
    • if the class of a data object is also a mode, you can interpret that to mean the data object is vector. As expected, the strings “matrix” and “data.frame” are returned when the class() function is applied to matrix and data frame objects, respectively.
    • install library: install.packages()

Importing Data into R

Working Directory

  • Get: getwd()
  • Set: setwd()
    • the setwd() function will NOT create a directory; if the directory supplied to the function does not exist, an error will be returned.
      • use /, or change the \ to \\ or \/ in the file path

Import Data Files

Asess Data

  • Entire Data: View()
  • Structure & Part of the Content: str()
  • Part of the Data: head()

Data File Types

  • ASCII (.txt)
  • Comma Separated Value (.csv)
  • Excel (.xlsx)
  • SAS (.sas7bdat)

Import txt Files

read.table()
Default Arguments

  • header=FALSE (TRUE: assign the first row as variable names)
  • sep=’ ‘(one ore more white spaces)
    • ‘\t’: tab
    • ‘,’: comma
    • others. eg. ‘/‘ or ‘@’
  • skip = 0 : number of lines of the data file to skip before beginning to read
  • nrows= -1: maximum number of lines to be read
  • na.strings=’NA’: a vector containing the missing value indicators. NA and blank spaces are automatically interpreted as missing values. Others must be listed(for example, na.strings=c(‘X’, 999))
  • stringsAsFactors=TRUE (FALSE: as character)

Import csv Files

read.csv()
read.csv() is essentially a wrapper for read.table().
Default Arguments

  • header = TRUE
  • sep=’,’
  • na.strings=’NA’
  • stringsAsFactors = TRUE

Import Excel Files

library(‘openxlsx’)
read.xlsx()
Default Arguments

  • colNames= TRUE (like header)
  • sheet=1
  • na.strings=’NA’

Converted Data Types

  • stores non-numeric data as character variable
  • able to detect date, monetary(\$) variable, and stores them as numeric (because variable formats can be stored; R convert date strings into a numeric value to represent this order )

Import SAS Files

library(‘haven’)
read_sas()

  • All SAS data files have the same structure, the function does NOT require customization.
  • Missing values in SAS data files are indicate by a period (.) for numeric variables, blank space ( ) for character variables.
  • The function returns a tibble data object.

Converted Data Types

  • stores non-numeric data as character variable
  • date as date variable
  • monetary(\$) as numeric variable

Managing Data in R

Once raw data files are read into R, the next step is to transform them into an “analysis ready” data set. It involves creating new variables within an existing data set or creating new data sets by combining multiple existing data sets.

  • column: variable
  • row: record

Work with Variables

  • add variables: D.frame$New.Var <- Temp

  • delete variables: D.frame$V.name <- NULL

  • continuous -> categorical: using logical indexing

  • flag variable: 1*(Logical Expression)
    ICU$High_Risk <- 1*(icu$sex="=" 'male'="" &="" icu$hra=""> 100)

  • aggregation

    • use mathematical operators
      ICU$Sev_Score1 <- ICU$SER + ICU$TYP + (ICU$SYS/100)

    • use apply(A, MARGIN=B, FUN=C, na.rm=FALSE)

      • MARGIN: 1–rows, 2–columns
      • na.rm=TRUE: does not assign missing values if any of the underlying variables are missing; instead, it assigns the sum of all available values

  • factor variable: factor()

    • values are sorted alpha-numerically
    • can be converted to numeric variable: as.numeric(factor(ICU$RACE))
    • table(Df$FV): acquire value counts for factor variable

  • date variable

    • can be converted to numeric variable: as.numeric(). R’s default origin date: 1970-01-01.
    • can do ‘-‘ operation, return ? days

  • identify missing values (NA): is.na()

    • count NA in all variables: apply(is.na(ICU), 2, sum)

    • NA are NOT allowed in indexing dataframe; when indexing vector it works as FALSE

    • comparison operators(<, >, !=) do NOT work: return NA

    • logical operators: if one value is missing and the other value is not, R will only return a missing value if the result of the Boolean comparison cannot be determined; if it can be determined, the appropriate logical value will be returned.

  • unique(): returns a vector, data frame or array with duplicate elements/rows removed

  • setdiff(a,b): returns a vector with the elements in a that are not in b

Concatenate Data Sets

Key

The variable or set of variables in the data set that can be used to uniquely identify individual records AND connect matching records across multiple data sets.

  • check duplicate key values: duplicated()

    • fromLast=FALSE, check sequentially (first occurrence of the key will not be returned)

    • vector: element-wise

    • dataframe: row-wise

Concatenation

A vertical combining of two data sets. Concatenation typically results in a data set with more records, but containing the same number of variables when compared to the individual datasets that were combined.

smartbind(data1, data2, …)

  • in library(‘gtools’)
  • assign NA for the records in the data set that does not contain the extra variable.

rbind()

  • return an error if the data frames being combined do not have the same number of columns OR if they do not have the same variable names.

Merge Data Sets

A horizontal combining of two data sets. Merging typically results in a data set with the same number of records, but containing more variables when compared to the individual datasets that were combined.

merge(Obj1, Obj2, by=A): merge Obj1 and Obj2 based on the key variable(s) listed in the vector A

  • all=FALSE

    • return only the records in the INTERSECTION of the sets of key values
    • TRUE: return the records in the UNION of the sets of key values

  • using wrong key: results in twice the number of records in the merged data set

  • merge by key variables with different names

    • using merge(Data1,Data7, by.x=c(‘Country’,’Year’), by.y=c(‘Nation’,’Time’)), output will keep x’s names
    • change the key var names in one of the data set

  • one-to-many merging: records with less keys are repeated across the matching records

    • complete.cases(): return a logical vector indicating records that have non-missing values for all variables

cbind()

  • merge based on row number; simple horizontal concatenation
  • if the number of records are different for two data sets, use re-cycling mechanism

Box Plot

  • outliers: distant observations
  • Interquartile range: IQR = Q3 - Q1
  • upper whisker = Q3 + 1.5*IQR
  • lower whisker = Q1 - 1.5*IQR
    In a sense, the box is like the body of the box plot and the whiskers are like its arms trying to reach the rest of the data.

Mean & Variance (Std)

  • Mean: the center of a data set
  • Deviation: the distance of an observation from its mean
  • Variance: the average squared distance from the mean
  • Standard deviation: the square root of the variance
    variance and Std are used as a means to an end, where the “end” is being able to accurately estimate the uncertainty associated with a sample statistic.
Name Sample Population
Mean \overline(x) μx
Deviation (偏差) $xi - \overline(x)$ -
Variance (方差) s2 = $\sum_{n=1}^N{\dfrac{xi - \overline(x}N}}$ σ2
Standard Deviation(标准差/均方差) s = 根号s2 σ

Robust Statistics

Robust statistics means extreme observations have little effect on their values
Robust: median, IQR
Unrobust: mean, standard deviation

Transforming Data

A transformation is a rescaling of the data using a function. Transformed data are sometimes easier to work with when applying statistical models because the transformed data are much less skewed and outliers are usually less extreme.
